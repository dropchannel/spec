# Conventions

This directory contains codified team practice for contributors to the DropChannel project. Conventions govern how work is done — branching, commit messages, pull requests, and similar workflow concerns.

Conventions are normative for contributors, not for implementations. They do not define what DropChannel is or how it must behave; they define how the people building it collaborate.

## Relationship to ADRs

Each convention here should trace back to an accepted ADR that motivated or ratified it. The ADR captures the decision and its reasoning; the convention document is the living reference contributors consult day-to-day. If a convention changes, update this document and record the change in a new ADR — do not amend the original.
